Foodelight Web Application

Roadmap

Frontend

public/

css/

Contains CSS files for styling the application.

js/

Contains JavaScript files for client-side functionality.

images/

Contains static images used in the application.

uploads/

Stores user-uploaded files like profile pictures.

videos/

Stores user-uploaded videos.

views/

admin/

Contains HTML templates for admin pages.

auth/

Contains HTML templates for login and signup pages.

restaurant/

Contains HTML templates for restaurant pages.

rider/

Contains HTML templates for rider pages.

user/

Contains HTML templates for user pages.

Backend

config/

auth.js

Handles authentication configurations like JWT secrets.

db.js

Contains database connection settings.

payment.js

Configures payment gateway settings.

redis.js

Configures Redis for caching.

server.js

Configures server settings like port and environment.

controllers/

adminController.js

Handles admin-related logic like managing users and restaurants.

authController.js

Manages user authentication and authorization.

restaurantController.js

Handles restaurant-related operations like menu management.

riderController.js

Manages rider-related logic like delivery assignments.

userController.js

Handles user-related operations like profile updates.

paymentController.js

Manages payment-related logic like processing transactions.

reviewController.js

Handles review-related operations like adding and fetching reviews.

analyticsController.js

Manages analytics-related logic like generating reports.

middleware/

authMiddleware.js

Validates user authentication for protected routes.

errorHandler.js

Handles and logs errors globally.

rateLimiter.js

Limits the number of requests from a single IP.

sanitize.js

Sanitizes user inputs to prevent XSS attacks.

cors.js

Configures CORS settings for cross-origin requests.

adminModel.js

Defines the schema for admin data.

Defines the schema for menu items.

orderModel.js

Defines the schema for orders.

paymentModel.js

Defines the schema for payment transactions.

restaurantModel.js

Defines the schema for restaurant data.

riderModel.js

Defines the schema for rider data.

transactionModel.js

Defines the schema for financial transactions.

userModel.js

Defines the schema for user data.

reviewModel.js

Defines the schema for reviews.

analyticsModel.js

Defines the schema for analytics data.

adminRoutes.js

Defines routes for admin-related operations.

authRoutes.js

Defines routes for authentication.

restaurantRoutes.js

Defines routes for restaurant-related operations.

riderRoutes.js

Defines routes for rider-related operations.

userRoutes.js

Defines routes for user-related operations.

paymentRoutes.js

Defines routes for payment-related operations.

reviewRoutes.js

Defines routes for review-related operations.

analyticsRoutes.js

Defines routes for analytics-related operations.

services/

authService.js

Handles authentication logic like token generation.

paymentService.js

Handles payment processing logic.

notificationService.js

Manages sending notifications to users.

searchService.js

Handles search functionality for restaurants and items.

utils/

notifications.js

Utility functions for sending notifications.

upload.js

Utility functions for handling file uploads.

validators.js

Utility functions for validating user inputs.

logger.js

Utility functions for logging application events.

Handles real-time communication using WebSockets.
